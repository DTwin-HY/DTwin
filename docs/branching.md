# Branching strategy

This document describes the branching strategy used for this project, and other related information.

## Project workflow

[GitHub flow](https://docs.github.com/en/get-started/using-github/github-flow) is the preferred workflow for this project.

## Branching and merges

* The preferred lifetime of branches is one to three days.
* Rebase before pull requests. Resolve conflicts in the feature branch.
* Preferably run tests and lint before merging to catch any issues.
